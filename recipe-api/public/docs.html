API Documentation

GET /recipes

Retrieve a paginated list of recipes. Supports full=true parameter to fetch complete details (ingredients, instructions) in the list view.

Response Example

{
  "recipes": [ ... ],
  "count": 150
}

GET /recipes/:id

Get full details for a specific recipe. Includes Just-In-Time (JIT) nutrition enrichment via FatSecret.

POST /recipes/enrich

Batch retrieve and hydrate multiple recipes by ID. Automatically fetches missing nutrition data for all requested items.

Payload

{ "ids": ["uuid-1", "uuid-2", ...] }

GET /search

Hybrid intelligent search.

Parameters

  • q: (string) Search term.
  • ingredients: (string) Filter by ingredients.
  • full: (boolean) If true, returns full recipe objects (ingredients, instructions) instead of summaries. Default false.

GET /health

Returns real-time system health statistics, including the number of verified vs. flagged recipes and recent automated audit logs.

Integration Guide

You can fetch data directly from your frontend or mobile app. No authentication is required for read operations.

// Example: Fetching recipes with 'chicken'
const res = await fetch('https://recipe-base.wearemachina.com/search?q=chicken');
const data = await res.json();
console.log(data.recipes);
